# 标题 title
*(面向普通用户，仅介绍语法与使用方式，不涉及实现细节)*

---

## 1. 概述

`title` 是 LiteML 目前唯一提供的语句，用来在生成的 HTML 中：

1. **在 `<body>` 中输出标题标签** (`<h1>`‑`<h6>`)。
2. **决定页面的 `<title>`**：页面 `<title>` 取 **第一次出现的 `title` 文本**（若没有 `title`，则使用默认文字 `LiteML`）。

> **注意**：所有 `title` 必须写在名为 `index` 的函数体内部；一个文件只能有这一个入口函数。

---

## 2. 基本语法

### 2.1 完整格式

```
title("标题文字");                     // level 默认为 1，生成 <h1>
title("标题文字", level=数字);          // 自定义层级，生成对应的 <hN>
```

- **关键字**：`title`（小写）
- **括号**：左圆括号 `(` 与右圆括号 `)` 必须配对。
- **字符串文字**：使用双引号包围，可使用 `\"` 与 `\\` 进行转义。
- **可选参数**：仅支持 `level`，须写成 `level=数字`，并使用逗号 `,` 与标题文字分隔。
- **结束符**：每条语句必须以分号 `;` 结束。

### 2.2 参数说明

| 参数 | 必须 | 类型 | 取值范围 | 默认值 | 说明 |
|------|------|------|----------|--------|------|
| 第一个参数（标题文字） | ✅ | 字符串 | 任意合法文本 | — | 要在页面中显示的文字 |
| `level` | ❌ | 整数 | 1 ~ 6 | 1 | 决定生成的 HTML 标题层级 `<h1>`‑`<h6>` |

> **层级映射**：`level=1` → `<h1>`、`level=2` → `<h2>` …… `level=6` → `<h6>`。

---

## 3. 示例

### 3.1 最简写法（默认 `<h1>`）

```ltml
index {
    title("欢迎使用 LiteML");
}
```

**生成的 HTML**（省略 `<!DOCTYPE>` 等头部，仅展示关键片段）：

```html
<head>
    <meta charset="utf-8">
    <meta name="generator" content="LiteML index">
    <title>欢迎使用 LiteML</title>
</head>
<body>
<h1>欢迎使用 LiteML</h1>
</body>
```

### 3.2 指定层级

```ltml
index {
    title("快速入门", level=2);
    title("第一步：创建文件", level=3);
    title("第二步：编译", level=3);
}
```

**HTML 关键片段**：

```html
<title>快速入门</title>
<body>
<h2>快速入门</h2>
<h3>第一步：创建文件</h3>
<h3>第二步：编译</h3>
</body>
```

> `<title>` 使用 **第一次出现** 的文本（本例为 “快速入门”），后面的 `title` 仅作为正文标题。

### 3.3 多个 `title`，只取第一条作 `<title>`

```ltml
index {
    title("主标题", level=1);
    title("章节 1", level=2);
    title("章节 2", level=2);
}
```

- `<title>` → **主标题**
- `<body>` → `<h1>主标题</h1>、`<h2>章节 1</h2>`、`<h2>章节 2</h2>`

### 3.4 没有任何 `title`

```ltml
index { }
```

- `<title>` → 默认 `LiteML`
- `<body>` → 空（无标题）

## 4. 错误提示（友好版）

| 错误情形 | 示例 | 友好提示 | 解决办法 |
|----------|------|----------|----------|
| **缺少分号** | `title("标题")` | “语句必须以 `;` 结尾”。 | 在末尾补上 `;` |
| **未闭合括号** | `title("标题", level=2` | “缺少右括号 `)`”。 | 在末尾补上 `)` |
| **`level` 超出范围** | `title("标题", level=9);` | “`level` 只能在 1~6 之间”。 | 将 `level` 改为 1‑6 之间的整数 |
| **`level` 参数名错误** | `title("标题", leval=2);` | “仅支持 `level` 参数”。 | 把 `leval` 改为 `level` |
| **`level` 参数缺等号** | `title("标题", level 2);` | “`level` 参数后必须有 `=`”。 | 使用 `level=2` 形式 |
| **`level` 后不是数字** | `title("标题", level=two);` | “`level` 必须是整数”。 | 使用数字，如 `level=2` |
| **标题文字不是字符串** | `title(123);` | “标题的第一个参数必须是字符串”。 | 用双引号包住文字，例如 `"123"` |
| **函数名不是 `index`** | `main { title("X"); }` | “只能使用唯一入口函数 `index`”。 | 把函数名改为 `index`（且只能出现一次） |
| **缺少 `title`** | `index {}` | “页面没有标题，默认 `<title>` 为 `LiteML`”。 | 如需自定义页面标题，添加至少一条 `title` 语句 |

> 所有错误在编译时都会指明出错的 **行号与列号**（字符位置），方便直接定位并修正。

---

## 5. 使用建议

1. **把最重要的文字放在第一条 `title`**，它会成为浏览器标签页的标题。
2. **层级递进**：建议遵循层级结构（1 → 2 → 3 …），保持页面结构清晰，利于 SEO 与可访问性。
3. **避免同一层级的标题连续出现**（除非真的需要），这样有助于阅读体验。
4. **使用转义**：如果标题里需要出现双引号 `"`，请写成 `\"`；如果需要反斜杠，请写成 `\\`。

---

## 6. 常见疑问（FAQ）

**Q1：可以在 `<body>` 中出现除 `title` 之外的其它语句吗？**
> 目前 LiteML 只实现了 `title`，其它语句尚未支持。将来会在 `Stmt` 枚举中加入更多功能。

**Q2：`title` 的文本可以写中文、emoji 吗？**
> 可以，只要放在双引号内部，编译器会原样输出（UTF‑8 编码）。

**Q3：如果同一行写多条 `title`，会怎样？**
> 语法允许，只要每条语句以 `;` 结束。它们会依次出现在 `<body>`，并且第一条仍然决定 `<title>`。

**Q4：`title` 文本会被 HTML 转义吗？**
> 编译器直接把文本写入标签中。若需要自行转义（如 `<`、`>`），请在字符串里使用对应的实体（`&lt;`、`&gt;`）或自行处理。

**Q5：`title` 的 `level` 参数可以省略吗？**
> 可以，默认值为 `1`（即 `<h1>`）。

**Q6：为什么必须使用 `index` 函数？**
> 这是一种约定，令编译器明确入口点。以后可能会支持多文件或路由，但现在只有一个入口函数。

## 7. 完整示例（包含注释）

```ltml
// demo.ltml
index {
    // 页面标题（同事也是浏览器标签页标题）
    title("LiteML 示例页面", level=1);

    // 二级标题
    title("快速入门", level=2);

    // 三级标题，展示多个同层级的写法
    title("创建文件", level=3);
    title("运行编译器", level=3);
}
```

**生成的 `demo.html`（关键片段）**：

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <meta name="generator" content="LiteML index">
    <title>LiteML 示例页面</title>
</head>
<body>
<h1>LiteML 示例页面</h1>
<h2>快速入门</h2>
<h3>创建文件</h3>
<h3>运行编译器</h3>
</body>
</html>
```


掌握上述规则后，即可用几行 LiteML 代码快速生成结构化、语义明确的 HTML 页面。！
